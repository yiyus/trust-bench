# Review: Issue #31 - Tables and plots (tables.py, plots.py)

## Summary

Tests-only (RED) review. `tests/reporting/test_tables.py` and `tests/reporting/test_plots.py`
define the expected surface for `trust_bench.reporting.tables` and
`trust_bench.reporting.plots`, exercised against `large_residual.backend_results()`. Both
modules fail to collect with `ModuleNotFoundError: No module named 'trust_bench.reporting'`,
confirmed by running the suite; the other 185 tests remain unaffected. The key-tuple order
used in the tests (`["rho", "backend"]`) matches `backend_results()`'s `(rho, backend.name)`
dict keys, and the metric columns asserted (`status`, `dist_to_opt`, `cost_gap`,
`grad_norm_final`, `n_feval`) all exist on `RunResult`. No correctness issues found.

## Findings

### Minor: `logx` behaviour is exercised but not asserted

`test_save_figure_writes_a_non_empty_image_file` calls `plot_metric_vs_sweep(..., logx=True)`
but no test checks that the x-axis actually ends up log-scaled. A regression that silently
drops the `logx` flag would not be caught.

Location: `tests/reporting/test_plots.py:24-33`

### Minor: single registered backend weakens the "one line per group" test

`BACKENDS` currently has one entry (`SciPyBackend`), so
`test_plot_metric_vs_sweep_draws_one_line_per_group` asserts `len(fig.axes[0].lines) == 1`.
It still catches a wrong-grouping bug (grouping by `rho` instead of `backend` would yield 10
lines, not 1), so it is not vacuous, but it will not distinguish "grouped correctly by
`backend`" from "grouped by anything with exactly one distinct value" until a second backend
is registered.

Location: `tests/reporting/test_plots.py:15-21`

### Note: untracked `prototype/` directory is unrelated to this issue

`prototype/harness.py`, `prototype/large_residual.py`, and `prototype/run_experiment.py` are
untracked scratch files exploring Gauss-Newton vs. full-Newton convergence; they are not
referenced by the reporting tests and are not mentioned in `docs/prs/31.md`. Not a blocker
since they are untracked, but worth confirming they are intentionally kept out of this PR
before committing.
