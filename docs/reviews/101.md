## Findings

No issues found.

### Note: `backend_results` is reused directly, unlike baseline's two-pass equivalent

`_large_residual_basin_rate_table` is built from the `rates` dict `backend_results()` already returns alongside `results`, requiring no extra solve pass. This is tighter than `_write_baseline`, whose `basin_df` comes from a separate `baseline.basin_rates()` call that re-runs every "standard" start `backend_results()` (analogous `large_residual` function) already computed once. Not a defect in this PR; noted for awareness only, since a reader comparing the two `_write_*` functions side by side might otherwise expect a symmetric second call.
