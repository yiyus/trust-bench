### Note: no round-trip coverage for `None` values on the other `Optional` scalar fields

`dist_to_opt`, `cost_gap`, `grad_norm_final`, `n_iter`, `n_feval`,
`n_jeval`, `n_heval`, and `trace` are all `Optional` per Section 4.3, but
only `timing=None` is exercised. The PR doc's rationale for singling out
`timing` (it is the only `Optional` field that is itself a nested
dataclass, so it is the only one a custom `from_dict` could plausibly
mishandle) is sound: plain scalars and lists round-trip through
`json.dumps`/`json.loads` as `None`/`null` without any custom handling, so
there is no equivalent failure mode to guard against. Not blocking; noting
only so the narrower scope is a deliberate reading of the criterion, not an
oversight.
