# Review: Issue #28 - Study: dimensionality

## Summary

Tests-only (RED) stage. This issue also adds `trust-krylov` to
`tests/backends/test_scipy_minimize.py` (the matrix-free method issue #17
deliberately left out until an acceptance criterion needed it) and
introduces a `slow` pytest marker plus a `Makefile`/`pyproject.toml`
change so `make test` skips the three expensive new tests while `make
coverage` (and CI, via `.github/workflows/ci.yml`) still runs them in
full. Both RED failure sets and the "169 tests unaffected" claim
reproduce exactly, including the "3 pass vacuously" detail (the same
disclosed pattern as issue #17). Independently reproduced every
numerical claim: `BFGS` exhausts its 200-iteration budget at `n=100`
landing at `dist_to_opt ~= 0.846`; a 2-step `L-BFGS-B` vs. dense-method
timing ratio at `n=1000` comfortably clears the test's 5x threshold in
both directions (`trust-exact` ~23x, `BFGS` ~9x locally); all three
matrix-free methods converge to `dist_to_opt < 1e-3` at every dimension
checked, including `n=1000`; and `trust-krylov`'s `show_options` output
confirms it exposes only `inexact`, no `ftol`/`xtol`/`gtol`, matching
`TOLERANCE_PARAMS["trust-krylov"] = frozenset()`. No issues found.

## Verification

- APL semantics: not applicable (pure Python/numpy/SciPy study, no APL
  expressions).
- `trust-krylov` capability claims confirmed directly:
  `scipy.optimize.show_options(solver="minimize", method="trust-krylov")`
  lists only `inexact`, no outer tolerance option; requires an analytic
  Jacobian/Hessian (`hess=problem.hessian`) and converges correctly when
  supplied.
- RED failure (backend tests) confirmed exactly: 6 failures
  (`ValueError`/`KeyError` on `trust-krylov`'s absence from
  `capabilities()`), plus 3 tests that pass vacuously
  (`test_unbounded_methods_reject_box_constraints`,
  `test_finite_difference_derivative_mode_is_rejected_...`,
  `test_analytic_only_hessian_methods_reject_...`, all parametrised on
  `trust-krylov`), consistent with issue #17's disclosed pattern.
- RED failure (new study) confirmed:
  `ModuleNotFoundError: No module named 'trust_bench.studies.dimensionality'`.
- Full suite minus the 6 affected backend-test instances and the new
  file: 169 passed, matching the PR doc's claim.
- `BFGS` failure at `n=100` independently reproduced: `dist_to_opt ~=
  0.846`, `nit=200` (budget exhausted), `success=False`.
- Dense-vs-matrix-free per-step cost ratio independently reproduced at
  `n=1000`, `max_iter=2`: `trust-exact`/`L-BFGS-B` ratio ~23x,
  `BFGS`/`L-BFGS-B` ratio ~9x, both comfortably above the test's `5x`
  threshold (local absolute times differ from the design notes', as
  expected for a machine-dependent measurement, but the qualitative
  and threshold-relevant claims hold).

  1000`: `Newton-CG`, `trust-krylov`, `L-BFGS-B` all converge to

  `8.7e-4` respectively).
- `slow` marker infrastructure confirmed: `Makefile`'s `test` target

  `.github/workflows/ci.yml` runs `make test`, `make lint`, and `make
  coverage` in sequence, so CI does still exercise the slow tests via
  the `coverage` step.
