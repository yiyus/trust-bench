# Review: Issue #98 - cli.py never surfaces baseline's basin-of-attraction rate

## Summary

Tests-only (RED) stage review. Both new/extended tests reproduce the described wiring gap exactly and fail for the stated reason. No issues found.

## Findings

### Note: reproduction confirmed exactly

Ran both tests directly against the unmodified `cli.py`. `test_run_report_writes_baselines_basin_of_attraction_rate_table` fails with `FileNotFoundError` reading `baseline_basin_rates.csv`, which `_write_baseline` never writes. `test_report_command_produces_every_milestone_artefact` fails with `AssertionError: baseline_basin_rates.csv` against the now-extended `_EXPECTED_TABLES` smoke check. Both match the PR doc precisely.

Confirmed the root cause directly in `trust_bench/cli.py:41-46`: `_write_baseline` calls only `baseline.standard_start_results()`; `baseline.basin_rates()` (`trust_bench/studies/baseline.py:23-35`) has no caller anywhere in `cli.py`. The function itself is complete and correct as it stands (per-`(problem_id, backend)` rate across every registered start), confirming this is a wiring gap, not a missing capability.

### Note: test design matches the data shape

`basin_rates()` returns one rate per `(problem_id, backend)`. With the default single-backend call (`BACKENDS = [SciPyBackend()]`), `test_run_report_writes_baselines_basin_of_attraction_rate_table`'s `len(df) == len(CANONICAL_PROBLEMS)` check is exactly right for that shape, and `basin_rate.between(0.0, 1.0)` correctly reflects `core/metrics.py:basin_rate`'s contract (a fraction in `[0, 1]`, or NaN for an empty start set - not exercised here since every canonical problem registers at least one start).

### Note: parallel gap correctly scoped out

`_write_large_residual` (`cli.py:49-55`) discards `large_residual.backend_results()`'s own basin-rate return value via `results, _ = ...` - the same class of gap. Confirmed issue #101 is filed and open for this, keeping the present fix narrowly scoped to `baseline`, consistent with this project's preference for single-concern changes.
