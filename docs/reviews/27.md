`'jac'` â€” there is no unscaled mode for `'lm'` through this parameter.

## Findings

### Major: `x_scale='jac'` is a no-op for method `'lm'`, breaking the nfev-comparison test

Per `scipy.optimize.least_squares`'s own docstring: "The default scaling
for each method (i.e. if `x_scale is None`) is as follows: ... For 'lm':
`x_scale == 'jac'`." Verified directly against the real problem family
(`trust_bench.problems.families.scaling.make`, bypassing the not-yet-wired
backend, same as the PR doc's own verification method): for `method="lm"`,
`x_scale=None` and `x_scale="jac"` give identical `nfev` at every scale in
`SCALES` (`2, 2, 4, 10, 17, 24`, both cases). At `s=_LARGE_DISPARITY=1e6`
this is `17 == 17`, so
`test_jac_scaling_needs_more_function_evaluations_at_large_scale_disparity`'s
`assert jac_scaled.n_feval > unscaled.n_feval` is false for `lm`, not
close-but-fragile the way issue #26's finding was: it is the same number.

This also explains a discrepancy in the design notes: `docs/prs/27.md`
states "`lm`: 2 -> 24 ... from `s=1` to `s=1e6`", but the actual measured
`nfev` for `lm` at `s=1e6` (either `x_scale`) is `17`; `24` is the value at
`s=1e8`, one step further than the design notes say. This is a symptom of
the same root confusion: the "unscaled vs. `jac`-scaled" comparison for
`lm` was never actually isolating two different behaviours, because
MINPACK's `lm` always self-scales via Jacobian column norms regardless of
what `x_scale` is passed.

The comparison is only meaningful for `trf`/`dogbox`, whose true default
(`x_scale == 1`, i.e. genuinely unscaled) differs from `x_scale='jac'`;
both of those show the claimed nfev increase and reproduce exactly.
Recommend excluding `'lm'` from
`test_jac_scaling_needs_more_function_evaluations_at_large_scale_disparity`'s
method loop (mirroring how
`test_jac_scaling_can_converge_to_the_wrong_answer_at_extreme_scale_disparity`
already scopes itself to `["trf", "dogbox"]`), and correcting the design
notes' `lm` figures or dropping `lm` from that paragraph's before/after
comparison entirely, since there is nothing to contrast for it.

Location: `tests/studies/test_scaling.py:23-35`
