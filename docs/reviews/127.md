## Findings

### Minor: adaptive `x_scale="jac"` rejection is pinned only for `lm`, not `BFGS`/`trust-exact`

`apl_backend.py`'s rejection of a string `x_scale` (`isinstance(config.x_scale, str)`) sits ahead of the per-method branching, so it is meant to apply uniformly to every method, not just `lm` - the design note frames it as a blanket statement ("no equivalent of scipy's adaptive x_scale... since Newton.aplo's own damping already gives an adaptive-scaling effect internally"), not an `lm`-specific one. `test_lm_rejects_adaptive_x_scale` is the only test exercising this path; a change that accidentally scoped the rejection to `lm` only (e.g. moved the check inside the `if method == "trust-exact"` branch, or added a `BFGS`-only carve-out) would still pass the full suite.
