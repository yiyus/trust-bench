# Review: Issue #26 - Study: bounded and constrained

## Summary

Tests-only (RED) stage. `tests/studies/test_bounded.py` covers the
scoped-down acceptance criterion (inactive, active-at-boundary, and
infeasible-start scenarios for SciPy's `trf`/`dogbox`, `trust`'s
Coleman-Li comparison deferred to Phase 8, consistent with issue #25's
precedent and the same, still-open Epic #34). The infeasible-start
`ValueError` claim and the three scenario/bounds constructions were
independently reproduced and check out. One finding was raised and has
since been fixed: two tests originally asserted exact floating-point
equality that `trf` does not satisfy under default tolerances; both now
use `pytest.approx(..., abs=1e-6)`, with the fix and its verification
recorded in `docs/prs/26.md`'s "Review response" section. No outstanding
issues.

## Findings

### Major (fixed): exact-equality assertions were not satisfied by `trf` under default tolerances

`test_inactive_bounds_converge_to_the_unconstrained_optimum` and
`test_active_at_boundary_bounds_converge_exactly_to_the_boundary`
originally asserted `outcome.x_final[i] == 0.0` / `== 0.5` directly.
Verified directly via `SciPyBackend.solve` with `RunConfig(max_iter=200,
bounds=...)` (no `tolerance` override, matching the pattern every other
study in this repo uses for its default sweep config):

- `dogbox` hit both targets exactly (`[0.0, 0.0]` and `[0.5, 0.0]`) in
  both scenarios.
- `trf` did not: `inactive` gave `[2.176e-16, -2.176e-16]`, and
  `active_at_boundary` gave `[0.5000000001164153, 0.0]`, under default
  tolerances. `trf` only reached the exact target at `ftol=xtol=gtol=
  1e-15`, essentially machine epsilon.

Fixed by switching both assertions to `pytest.approx(..., abs=1e-6)`
(`tests/studies/test_bounded.py:6,25-26,35-36`), comfortably covering
both methods' actual deviations (`~1e-16` and `~1e-10`) without relying
on a tolerance one order of magnitude above machine epsilon. Re-verified
directly: `1e-6` gives ample margin over both methods' measured
deviations in both scenarios.

Location: `tests/studies/test_bounded.py:19-36`
