# Review: Issue #8 - Problem dataclass and registry (problem.py, registry.py)

## Summary

Tests-only (RED) stage review. `tests/unit/test_problem.py` and
`tests/unit/test_registry.py` cover both acceptance criteria plus the
`frozen=True` contract from Section 4.1 of the design doc. The RED failure
(`ModuleNotFoundError: No module named 'trust_bench.core.problem'`) is
confirmed by running the suite. One Minor gap in how precisely the
callable-rejection test pins down the acceptance criterion; no other issues
found.

## Findings

### Minor: the callable-rejection test only exercises `residual=None`, not a general non-callable

AC2 says "registering a `Problem` without a `residual` callable is
rejected." `test_register_rejects_a_problem_without_a_callable_residual`
only constructs the fixture with `residual=None`. An implementation that
checks `residual is None` (rather than `not callable(residual)`) would pass
this test while still accepting, say, `residual="not a function"` or
`residual=42`, values the acceptance criterion's wording implies should
also be rejected. The PR doc's design notes say "the callable check lives in
`Registry.register()`", which suggests the intended implementation is a
`callable()` check, but the test as written does not force that distinction
over a narrower `is None` check.

Recommendation: add a second case (e.g. `residual="not callable"`) to the
same test, or a second test, asserting `TypeError` for a non-`None`,
non-callable value. Not blocking; the current test is a correct subset of
the criterion, just not the full statement of it.

Location: `tests/unit/test_registry.py:40-44`

### Note: no test for `Registry.get()` on an unregistered id

Neither acceptance criterion requires this, and `by_tag`'s absence case
(the untagged fixture) already exercises one flavour of "not found."
Flagging only so a `get()` miss (`KeyError` vs `None` vs something else) is
a deliberate choice once `registry.py` exists, not an accident of whichever
branch gets written first.

Location: `tests/unit/test_registry.py`
