## Implementation review

### Summary

`derivative_source.sweep` and `scaling.sweep` gain `ill_conditioning.
sweep`'s exact skip-guard (`if method not in supported: continue`).
`scaling.sweep` additionally wraps its `run()` call in `try/except
ValueError`, a second fix found while verifying the acceptance criterion
against real `APLBackend` rather than only the stub-backed RED tests:
`"lm"` is a method `trust-apl` supports, but `x_scale` is a config field
within that method it rejects, and `MethodCapabilities` has no
declarative field to skip on for that case in advance. This matches
`bounded.py`'s own `try/except ValueError` precedent exactly (verified
by reading `trust_bench/studies/bounded.py`), and `results_to_dataframe`
already renders any `BaseException` value as a visible `ERROR` row
(verified by reading `trust_bench/reporting/tables.py`), so no
reporting-side change was needed. The self-found gap is covered by a new
test (`_LMWithoutXScaleBackend` /
`test_sweep_records_a_raised_value_error_as_an_outcome_instead_of_crashing`)
that fails before the fix and passes after. No issues found.

### Findings

No issues found.

### Verification

- Architectural conformity: `derivative_source.py`'s skip-guard matches
  `ill_conditioning.py`'s pattern verbatim. `scaling.py`'s additional
  `try/except ValueError` matches `bounded.py`'s pattern verbatim.
- `_check_backend_coverage` (from #77): confirmed it only requires a
  backend's name to appear at least once in the output `DataFrame`
  (`trust_bench/cli.py:34-36`), which an `ERROR` row still satisfies, so
  wrapping instead of skipping the `x_scale` case doesn't reintroduce a
  silent-empty-result risk.
- Comment hygiene: no GitHub issue/PR IDs in source comments. Docstrings
  and the new stub-backend class comments state invariants and
  constraints (e.g. why there is no declarative capability for
  `x_scale`), not narrative.
- Tests: full suite run. `.venv/bin/pytest -m "not slow"`: 230 passed,
  28 deselected (3 new, matching the PR doc: the two RED-reviewed
  skip-guard tests plus the self-found `x_scale` test).
- Lint: `.venv/bin/ruff check .` clean.
- Real-backend verification claims in the PR doc (real `APLBackend` run,
  `run_report` end-to-end with `scipy`+`trust-apl`) were not
  independently re-run in this review; taken on the author's report,
  consistent with the code paths read above.
