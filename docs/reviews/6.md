# Review: Issue #6 - Basin-of-attraction rate aggregator

## Summary

Tests-only (RED) stage review. `tests/unit/test_metrics.py` adds two tests
for `basin_rate()` that cover both acceptance criteria in the issue plus an
empty-input guard. The hand-computed fixture fraction was verified
independently, and the RED failure
(`ImportError: cannot import name 'basin_rate'`) is confirmed by running the
suite. No issues found.

## Findings

No issues found.

### Note: `basin_rate()` takes plain distances, not `RunResult`s

The issue text says "given a set of `RunResult`s", but `RunResult` is
Phase 3's deliverable (issue #12), scheduled after Phase 1 in the plan's own
ordering (`docs/plans/trust-bench.md`, Section 10). The PR doc documents this
explicitly and the chosen signature (`Iterable[float | None]`) avoids a
forward dependency on a module that does not exist yet, consistent with the
plan's "language-agnostic core" principle (Section 2: pure functions in
`metrics.py` carry no cross-module type dependencies). No action needed; this
is a reasonable resolution of a genuine scheduling gap between the issue text
and the phase plan, not a deviation to flag.

Location: `docs/prs/6.md`, "Design notes: `RunResult` does not exist yet"

## Verification

- APL semantics: not applicable; this module is pure Python/NumPy, not APL.
- Fixture numerics: independently recomputed the hand-computed fraction.
  `[1e-9, 0.3, 1e-10, None, 2.5]` with `tol=1e-6`: two values
  (`1e-9`, `1e-10`) are `<= tol`; `None` and the two out-of-tolerance values
  count in the denominator but not the numerator, giving `2/5 = 0.4`,
  matching the test's assertion.
- Empty-set guard: reasonable by analogy with `order()`/`rate()`'s existing
  too-few-points NaN guards, and necessary to avoid a `0/0` division; not a
  literal reading of either acceptance criterion but the PR doc says so
  explicitly rather than presenting it as directly required.
- Comment hygiene: clean. No GitHub issue/PR IDs in
  `tests/unit/test_metrics.py`.
- Tests: RED confirmed. `pytest tests/unit/test_metrics.py` fails at
  collection with `ImportError: cannot import name 'basin_rate' from
  'trust_bench.core.metrics'`, the same failure class as issues #2, #3, and
  #5.

## Recommendation

Approve.
