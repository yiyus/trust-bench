# PR notes: Issue #2 - Package skeleton, lint, coverage and CI workflow

## Summary

Adds the `trust_bench` package skeleton, a `Makefile` with `test`, `lint`,
and `coverage` targets, and a CI workflow that runs those targets on every
push and pull request. Part of Epic #1 (Phase 0), see
`docs/plans/trust-bench.md`, Section 10 (Phase 0).

**Confirmed failures**

All six tests fail today because the package, `Makefile`, and CI workflow do
not exist yet:

- `ModuleNotFoundError: No module named 'trust_bench'`
- `make: *** No rule to make target 'test'.` (same for `lint`, `coverage`)
- `FileNotFoundError: [Errno 2] No such file or directory: '.github/workflows/ci.yml'`

**Design notes**

- The Makefile-target tests run `make -n <target>` (dry run) rather than the
  target itself: `make test` invokes pytest and `make coverage` invokes
  pytest under coverage, so calling either from inside a pytest test would
  recurse into the suite that contains the test.
- The CI-workflow tests check the checked-in YAML by text pattern rather than
  by triggering a run: GitHub Actions can only be exercised after a push, so
  this surface confirms the workflow file's shape, not that Actions accepts
  and runs it. That gap is closed once this branch is pushed and the PR's
  checks run for real.

**Anticipated implementation order**

1. `pyproject.toml`: package metadata, `pytest`/`ruff`/`coverage` as dev
   dependencies.
2. `trust_bench/__init__.py`: minimal package, no logic yet.
3. `Makefile`: `test`, `lint`, `coverage` targets.
4. `.github/workflows/ci.yml`: runs the same three targets on `push` and
   `pull_request`.

## GREEN cycle (implementation)

**Changes**

- `pyproject.toml`: setuptools build, `trust_bench` package discovery,
  `dev` extra (`pytest`, `ruff`, `coverage`), `ruff` configured with
  `line-length = 100` and `exclude = ["prototype"]`.
- `trust_bench/__init__.py`: empty; the package has no logic yet.
- `Makefile`: `test` runs `pytest`; `lint` runs `ruff check .`; `coverage`
  runs `coverage run -m pytest` followed by `coverage report`.
- `.github/workflows/ci.yml`: on `push` and `pull_request`, checks out the
  repo, sets up Python 3.11, runs `pip install -e .[dev]`, then `make test`,
  `make lint`, `make coverage`.
- `.gitignore`: added `*.egg-info/`, the build artefact `pip install -e .`
  leaves behind.

**Ruff scope**

`ruff check .` initially flagged seven pre-existing style violations under
`prototype/` (unused import, multiple statements per line). `prototype/` is
the throwaway experiment `docs/plans/trust-bench.md` Section 1.1 keeps for
reference only, not part of the maintained package, so it is excluded from
lint rather than fixed.

**Verification**

- `pip install -e .[dev]` succeeds from a clean checkout.
- `make test`: 6 passed.
- `make lint`: all checks passed (after excluding `prototype/`).
- `make coverage`: 6 passed, 100% coverage on `trust_bench/__init__.py` and
  the test file itself.
