# PR notes: Issue #5 - Convergence order and rate estimators

## Summary

Adds `trust_bench.core.metrics`: `order()` and `rate()`, pure functions that
estimate empirical convergence order and linear rate from an error
sequence. Part of Epic #4 (Phase 1), see `docs/plans/trust-bench.md`,
Section 10 (Phase 1); re-derived from `prototype/harness.py`'s
`convergence_order`/`linear_rate` under TDD, not lifted verbatim.

**Design notes**

- `prototype/harness.py`'s `convergence_order` hits the exact scenario
  `run_experiment.py:80-83` describes: "quadratic hits eps in ~3 steps" in
  float64, starting from a typical near-root perturbation. A literal
  `e_0=0.1`-style sequence still leaves 4 points above the `1e-12` floor
  (verified numerically), which is not few enough to trigger the guard. The
  RED/GREEN fixtures pick deliberately: `e_0=0.9` for the order test (9
  points survive the floor, comfortably above the guard), and `e_0=1e-3` for
  the NaN test (only 2 points survive), reproducing the prototype's failure
  mode on purpose rather than by accident of a specific starting value.
- The estimator's own guard threshold (implementation detail, not an
  acceptance criterion) requires at least 4 points above the floor before
  attempting a median-of-orders estimate, one consolidated condition rather
  than the prototype's two separate checks (`len(e) < 3` then
  `len(orders) < 2`), since for this formula the two are equivalent: 4
  surviving points always yield exactly 2 order values.
- `rate()` is given the same floor-filtering shape as `order()` for
  consistency, but its own too-few-points guard is not covered by a
  dedicated test: issue #5's acceptance criteria only require the NaN guard
  for `order`.
- This is the first module needing NumPy; `numpy` moves from absent to a
  real runtime dependency in `pyproject.toml` (not just a dev tool), since
  `metrics.py`'s vectorised log-ratio computation needs it directly.

**Anticipated implementation order**

1. `pyproject.toml`: add `numpy` to `[project] dependencies`.
2. `trust_bench/core/metrics.py`: `order()`, `rate()`.
