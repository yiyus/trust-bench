# PR notes: Issue #133 - Bump trust to 05aa864 (Converged-vs-Stalled fix)

## Summary

Bumps the vendored `trust` submodule to `05aa864`, closing out a
status-classification regression found while investigating
`scaling.csv`'s `scale=1e6` row: a genuinely converged run (`cost_final
≈1.95e-16`, far below `tolc`) whose gradient norm sits just over
`Result.gtol=1e-2`'s heuristic cutoff was reported `STALLED` instead of
`CONVERGED`, because `StalledByEscalation`/`StalledByPrecision` never
checked `~Converged` before firing.

Fixed entirely upstream, across four commits landed since the vendored
`d43a87d`:

- `5acffe4` ("do not report convergence as stalled"): bakes
  `tolc≤cost` into `StalledByEscalation`, `StalledByPrecision`, and
  `Stalled` itself, so none of the three can fire on an already-converged
  run regardless of caller ordering.
- `bd62a45` ("fix Stalled"): the same commit's own `Stalled` change used
  `∨` instead of `∧` (`(tolc≤cost)∨tolr>rel`), broadening it to fire on
  *any* non-convergence rather than specifically the relative-change
  stall it's meant to describe (confirmed directly: a `MaxIterations`
  case with `rel` nowhere near `tolr` reported `Stalled=1`). Fixed to
  `∧`.
- `ae359a0` ("not stalled when rel<0"): a second, independent bug
  surfaced while re-verifying the previous fix - `rel=-1` is
  `Newton.aplo`'s own "not yet computed" sentinel for a trial not yet
  measured, and `Result.apln`'s externally-recomputed `tolr>rel` check
  doesn't guard against it the way `Newton.aplo`'s own internal
  termination check already does (`(r≥0)∧tr>r`). Any run whose last
  recorded trial happened to be a domain-error trial left `rel` at the
  sentinel, making `tolr>rel` trivially true. Fixed by adding `rel≥0` to
  `Stalled`, and refactoring `StalledByEscalation`/`StalledByPrecision`
  to delegate to `Stalled` directly rather than duplicating the same
  `tolc`/`tolr`/`rel` logic (so this class of bug can't reappear
  independently in the three predicates that share it).
- `4c85139`/`05aa864`: a stray typo character left in `Result.apln` by
  an editor, and additional upstream test coverage for the `Result`
  namespace.

`solve.dyalog` only calls `StalledByEscalation`/`StalledByPrecision`,
never bare `Stalled` - so the `Stalled`-broadening bug never corrupted
this project's own report, but both derived predicates share the fixed
logic now via the same delegation, closing the gap for any future
caller (including `trust-bench` itself) that might call `Result.Stalled`
directly.

No `trust-bench`-side code changes: `solve.dyalog`'s classification
chain already calls the right predicates; the fix is entirely in what
those predicates compute.
