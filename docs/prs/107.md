# PR notes: Issue #107 - Add a plot for the typical-problems study

## Summary

#104 added the `typical` study but deliberately scoped out a plot,
since its own acceptance criteria only asked for the study and its
table. #97's own plot list didn't include `typical` since that study
didn't exist yet when #97 landed.

`typical`'s data is categorical - one row per `(problem_id, method,
backend)`, no continuous sweep axis like `kappa`/`n`/`scale`/`fraction`
- so `plot_metric_vs_sweep` doesn't apply directly; closer in shape to
`bounded`'s own excluded case, but unlike `bounded`, `typical`'s
`problem_id` axis is exactly the kind of small, fixed set of
categories a grouped bar chart is built for. Adds a new
`plot_metric_by_category`: one bar cluster per `problem_id`, one bar
per `(method, backend)` combination within each cluster, mirroring
`plot_metric_vs_sweep`'s own non-`CONVERGED` marker (hatching instead
of a hollow point, since there's no line to overlay a marker onto).

**Design notes**

*A grouped bar chart, not a reworked `plot_metric_vs_sweep`.* Verified
directly against the real `typical.csv` data (both the default
scipy-only sweep and one including `trust-apl`, whose method count
differs per backend - 9 for scipy, 3 for `trust-apl`): the function
handles a `group` combination that's absent for some categories
without crashing, reindexing to a gap in that cluster rather than
raising.

*`dist_to_opt` on a log y-axis*, matching `ill_conditioning.png`'s own
choice for the same reason: values span many orders of magnitude
(from near-`1e-15` for a converged fit to `>1e9` for `trust-apl`'s
`trust-exact` diverging on `noisy_expdec`/`gaussian_peak`, confirmed in
#104's own review cycle), and a linear scale would make every
converged bar invisible next to a diverged one.

## GREEN cycle (implementation)

`reporting/plots.py`: new `plot_metric_by_category(df, category, y,
group, logy=False, status_col=None)` - a grouped bar chart, one
cluster per unique `category` value, one bar per `group` combination
within each cluster; a bar whose `status_col` value isn't `"CONVERGED"`
is hatched. `cli.py`'s `_write_typical` now also plots `dist_to_opt`
vs `problem_id`, grouped by `method`+`backend`, log-y, with the status
overlay, and saves `typical.png`.

Verified end-to-end (direct `run_report` call, matching the RED-stage
verification approach): `typical.png` writes a real, non-empty image
alongside `typical.csv`, for both the default scipy-only sweep and one
including `trust-apl` (whose method count differs per backend, handled
without crashing).
