# PR notes: Issue #22 - Study: baseline correctness and basin-of-attraction rate

## Summary

Adds the first capability study (`docs/plans/trust-bench.md`, Section 9
item 1): the canonical set at its standard starts, run across every
registered backend, plus each problem's basin-of-attraction rate across
every one of its registered starts (not just "standard"), including
Rosenbrock's deliberately hard `far` start. Part of Epic #21 (Phase 6).

Introduces `trust_bench.backends.BACKENDS`, the production analogue of
the test-only `tests/backends/all_backends.py` list: the real backends a
study iterates over (currently just `SciPyBackend`), as opposed to the
test-only list that also carries `ToyGradientDescentBackend` to exercise
the Phase-4 contract. Studies are production code and must not depend on
a test double.

**Design notes**

Per Phase 6's own instruction ("assert shape, not exact numbers"), the
convergence/range assertions are shape-level: `CONVERGED` status and a
rate in `[0, 1]`, not specific numeric targets. Confirmed directly that
every canonical problem converges from every registered start with
`SciPyBackend`'s `lm` method at `max_iter=200` (including Rosenbrock's
`far` start), so a bare rate-in-range assertion cannot distinguish
"evaluated `far` and it happened to also converge" from "only ever
evaluated `standard`". `test_basin_rate_evaluates_every_registered_
start_not_only_standard` closes that gap directly: it spies on calls to
`runner.run` and asserts `far` is among the starts actually passed for
Rosenbrock, rather than inferring it from the rate's magnitude.
