# PR notes: Issue #86 - Document and test tolerance comparability between scipy and trust-apl

## Summary

Every existing study constructs its `RunConfig` with `tolerance=None`, so
each backend silently falls back to its own native default: scipy's
`least_squares` to `ftol=xtol=gtol=1e-8`, `trust-apl` to `âŽ•CT` (measured
`1e-14`). `solve.dyalog` already maps a single `RunConfig.tolerance` onto
both of `trust`'s Newton-loop stopping parameters (`tolc`, the absolute
cost floor, and `tolr`, the relative parameter/cost-change stall-detection
metric) - this mapping was added incidentally by #78 for bounds support,
but has no dedicated test and is undocumented. This issue documents that
mapping, adds a `tolerance_comparability` study that pins one explicit
tolerance value across backends so `dist_to_opt`/`grad_norm_final` can be
read side by side, and adds a behavioural test proving `APLBackend`'s
solve precision genuinely responds to `RunConfig.tolerance`.

Empirically confirmed before writing any test (`dyalogscript`,
`rosenbrock.PROBLEM`, method `"lm"`): scipy's `lm` reaches
`dist_to_opt == 0.0` at every tolerance from `1e-2` to `1e-14`, unaffected
across that whole range, while `trust-apl` degrades smoothly from `0.078`
at `1e-2` down to a floor of `1.55e-7` from `1e-10` onward. The two
backends' `tolerance` parameters are not semantically equivalent stopping
criteria: an identical numeric value does not imply comparable precision,
confirming the issue's own premise directly rather than assuming it.

## GREEN cycle (implementation)

`trust_bench/studies/tolerance_comparability.py` (new): `sweep()` runs
every canonical problem through `"lm"` on every backend at one explicit
`RunConfig.tolerance` (default `1e-8`, scipy's own native default,
chosen as a natural common value rather than an arbitrary one). Mirrors
`baseline.py`'s structure. Not wired into `cli.py`'s `STUDIES`: nothing
in the acceptance criteria or RED tests calls for a CLI-facing report
artefact, and the module is usable standalone or from a script; adding
CLI plumbing without a test driving it would be scope beyond this
issue.

`docs/methodology.md` (new): documents scipy's three-parameter
(`ftol`/`xtol`/`gtol`) mapping and `trust-apl`'s two-parameter
(`tolc`/`tolr`) stall-detection mapping side by side, states why an
equal numeric `tolerance` is not an equal stopping test, and gives the
concrete rosenbrock figures confirmed during the RED stage as a live
example. Cross-referenced from `tolerance_comparability.py`'s
docstring.

Verified: full suite (`dyalogscript` available) - 304 passed (up from
