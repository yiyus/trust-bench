# PR notes: Issue #42 - compare.py: Tier-1 regression vs Tier-3 drift classification

Part of #41 (Epic: Phase 10, Longitudinal tooling).

## Acceptance criteria

- a synthetic Tier-1 change between two result sets is classified as a regression
- a synthetic Tier-3 change is classified as drift, grouped by machine_fingerprint and backend_version

## Cycle 1: RED

`tests/reporting/test_compare.py` drives out `trust_bench/reporting/compare.py`,
not yet written. Tests build baseline/candidate result sets with
`storage.append`/`storage.load` (the same round trip `trust-bench compare`
will use against real `results/*.jsonl` files), matching how
`tests/unit/test_storage.py` and `tests/reporting/test_tables.py` already
construct `RunResult`s.

Covers, per Section 6 and Section 8 of `docs/plans/trust-bench.md`:

- Identical baseline/candidate results classify as `STABLE`.
- A changed Tier-1 metric (`dist_to_opt`) classifies as `REGRESSION`, and the
  changed metric's name is reported.
- A changed `status` (e.g. `CONVERGED` to `STALLED`) also classifies as
  `REGRESSION`: it is Tier-1's other stability guarantee, not only the
  numeric fields.
- A Tier-1 metric change within a tolerance floor (1e-12, well below any
  floating-point noise this project measures elsewhere) does not classify
  as a regression, so `compare` doesn't flag solver-internal rounding
  noise as a behavioural change.
- A changed Tier-3 metric (`timing.median`) with no Tier-1 change
  classifies as `drift`, not `regression` - Tier-3 is expected to move.
- A drifted row carries `machine_fingerprint` and `backend_version` (read
  from the candidate's provenance), so a downstream summary can group by
  them.
- When both a Tier-1 and a Tier-3 change occur on the same row,
  classification is `REGRESSION`: a genuine behavioural regression is
  reported even if timing also moved, rather than being masked by drift.
- `drift_summary` groups classified `drift` rows by `machine_fingerprint`
  and `backend_version` and counts them, giving the CLI's `compare`
  command the grouped view Section 8 specifies.
