# PR notes: Issue #115 - Bump vendored trust submodule: fixes trust-exact's indefinite-Hessian fragility

## Summary

The vendored `trust` submodule (`backends_ext/apl/trust`) has three new upstream commits: `0440f39` (fix Hessian in the Powell test), `05f9010` (reject all steps with predicted negative growth), `fa2e04d` (additional tests for indefinite Hessians and monotonic growth).

`05f9010` changes `Newton.aplo`'s step-acceptance check from `(⎕CT>c0)∧⎕CT<c1` to `(0>c0)∨(⎕CT>c0)∧⎕CT<c1`: a step whose quadratic model predicts a negative error decrement (`c0<0`, which happens when damping isn't enough to counteract an indefinite Hessian) is now rejected outright, rather than only when the actual outcome also failed to improve. This is exactly the mechanism #110's "Capability boundary: trust-exact's indefinite-Hessian fragility" documented as a real, measured limitation.

## RED cycle (submodule bump alone)

**Confirmed RED**

Bumping the submodule pointer alone (no other change) breaks exactly one test: `tests/studies/test_typical_study_apl.py::test_trust_apls_trust_exact_diverges_where_the_true_hessian_is_indefinite_away_from_the_optimum` now fails, because `trust-apl`'s `trust-exact` reports `CONVERGED` on `noisy_expdec` where the test still asserts `MAX_ITER`.

```
AssertionError: noisy_expdec
assert <RunStatus.CONVERGED: 'CONVERGED'> is <RunStatus.MAX_ITER: 'MAX_ITER'>
```

Confirmed directly (both problems, both statuses and precision): `noisy_expdec` now reaches `CONVERGED` (previously `MAX_ITER`, `dist_to_opt≈1012`); `gaussian_peak` now reaches `CONVERGED` (previously `MAX_ITER`, `dist_to_opt≈1.69e9`). Every other test in the affected files - `test_typical_study_apl.py`'s other test, `test_apl_difficulty_families.py` (including #95's `STALLED`-classification regression test at `ill_conditioned(kappa=1e7)`), `test_ill_conditioning.py`, `test_dimensionality_apl.py` - 18 tests total, all still pass: `ill_conditioning`/`dimensionality`/BFGS behaviour is unaffected (`ill_conditioned`'s Hessian is never indefinite; BFGS never uses the Hessian at all).

This is exactly the scenario the failing test's own comment anticipated: "a future change to trust's engine that fixes or worsens this is a deliberate test change, not silent drift."

**Design notes**

*Update the test, don't delete it.* The test's job was always to guard against silent drift in this specific behaviour; now that the behaviour has changed for the better, the test should assert the new, correct outcome (`CONVERGED`, small `dist_to_opt`) so a future regression back to the old failure mode is still caught.

*`docs/methodology.md`'s "Capability boundary: trust-exact's indefinite-Hessian fragility" section needs correcting*, not deleting: `dimensionality`'s mild, harmless indefiniteness is still a real, unaffected property of that problem's Hessian (confirmed unchanged above), but the severe, fatal case (`noisy_expdec`/`gaussian_peak`) that made this a genuine capability boundary is now resolved upstream.
