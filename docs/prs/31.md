# PR notes: Issue #31 - Tables and plots (tables.py, plots.py)

## Summary

Adds `trust_bench/reporting/tables.py` and `trust_bench/reporting/
plots.py`: generic building blocks that turn a study's `sweep()` result
dict into a comparison table (`pandas.DataFrame`) and a plot
(`matplotlib.Figure`), plus helpers to write both to disk as artefacts.
Part of Epic #30 (Phase 7), the milestone phase ("full Python-only
pipeline is shippable here").

Adds `matplotlib` as a project dependency (`pyproject.toml`); no other
module in the codebase needed plotting before this issue.

**Design notes**

Built generically (`key_names` supplied by the caller) rather than
tailored to one study, since every study's `sweep()` dict has a
different key-tuple arity (`baseline`: `(problem_id, backend)`;
`ill_conditioning`: `(kappa, method, backend)`; `derivative_source`:
`(problem_id, method, mode, backend)`; etc.) — the same shape choice
`sweep()` itself already makes in every study module. Exercised end to
end against `large_residual.backend_results()`, a real study with a
real, already-reviewed dataset, rather than synthetic fixture data.

`plots.py` calls `matplotlib.use("Agg")` at module import time,
unconditionally: this is what makes "headless (no display)" true by
construction rather than by the test environment happening to lack a
display. Deliberately does not attempt bounded.py-style result dicts
that mix `RunResult` and a raised `ValueError` in the same sweep (issue
#26): `results_to_dataframe` assumes every value is a `RunResult`, and
handling that mix is deferred to whichever future issue (likely the
`report` CLI command, issue #33) actually builds a report for that
specific study.

called `large_residual.backend_results()` fresh in every single test

`make test` went from ~9-13s to ~31s. Switched both files to a
module-scoped `pytest.fixture` so the sweep runs once per file (twice
total) instead of once per test; `make test` now sits at ~15s, close to
its pre-issue baseline. No `slow` marker needed.
