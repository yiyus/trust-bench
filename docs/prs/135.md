# PR notes: Issue #135 - Restructure the HTML report

## Summary

Restructures `build_html_report` into three tiers: headline artefacts first,
themed grouping for the rest, and a fixed backend colour scheme applied
consistently to both tables and plots. Status values render as coloured
pills. Folds in #138's `RunResult.timing` data as new columns, displayed as
"median ± mad" with units, alongside a caveat that it supports no bare
cross-language claim.

Non-goals (explicit, per the issue): `scalar_cost`'s exclusion from the
report is already handled by #130, not touched here; no Phase 10
longitudinal comparison.

## Cycle 1: RED

### Tests

- `tests/reporting/test_colors.py` (new): a `trust_bench.reporting.colors`
  module doesn't exist yet. Tests define its contract: `BACKEND_COLORS`
  covering `scipy`/`trust-apl`, `STATUS_COLORS` covering every `RunStatus`
  value plus the reporting-layer-only `"UNSUPPORTED"` string (treated as
  neutral, not a failure colour), `backend_color()`/`status_color()`
  lookups with a safe fallback for unregistered names.
- `tests/reporting/test_plots.py` (extended): three new tests assert that
  `plot_metric_vs_sweep`, `plot_metric_by_category`, and
  `plot_capability_frontier` all draw the same backend using the same
  fixed colour from `colors.backend_color()`, regardless of group
  ordering or which other groups are present in a given call - matplotlib's
  own default colour cycling currently assigns backend colours
  inconsistently across different plots.
- `tests/reporting/test_tables.py` (extended): `results_to_dataframe` must
  flatten a `RunResult.timing` (a `TimingStats`-or-`None` object) into five
  new columns (`timing_median`, `timing_mad`, `timing_n_reps`,
  `timing_warmup`, `timing_thread_count`), blank when `timing` is `None`
  (the common case, since `measure_timing` defaults to `False`) or on an
  `UNSUPPORTED` row.
- `tests/reporting/test_html_report.py` (rewritten): the naive glob-based
  tests are replaced with tests asserting the three-tier structure -
  headline artefacts (`parity_scatter`, `capability_frontier`,
  `capability_matrix`) rendered before any themed study; the three theme
  groups (*Common problems*, *Difficulty axes*, *Special capabilities*)
  each rendered as a heading before its member studies; one-line captions
  present for at least one study per tier; a study absent from the output
  directory (e.g. skipped for a coverage gap) omitted without crashing;
  status values rendered via `colors.status_color()`, not as bare text; a
  fixed backend-colour legend always shown, regardless of which backends
  the report's data actually contains; timing values displayed as
  "median ± mad" with units; a timing caveat shown only when at least one
  study has timing data.

## Cycle 1: GREEN

### Implementation

- `trust_bench/reporting/colors.py` (new): `BACKEND_COLORS`
  (`scipy`/`trust-apl`), `STATUS_COLORS` (every `RunStatus` value plus
  `UNSUPPORTED`, coloured neutral rather than as a failure),
  `backend_color()`/`status_color()` with a fallback grey for an
  unregistered name.
- `trust_bench/reporting/plots.py`: `plot_metric_vs_sweep`,
  `plot_metric_by_category`, and `plot_capability_frontier` now resolve a
  line's or bar's colour through `colors.backend_color()` whenever
  `"backend"` is part of the group being plotted (bare column or one
  entry of a multi-column group), so the same backend reads as the same
  colour across every plot regardless of what else is being grouped by
  or in what order. `plot_parity_scatter` and `plot_capability_matrix` are
  unchanged: neither groups by backend identity.
- `trust_bench/reporting/tables.py`: `results_to_dataframe` flattens
  `RunResult.timing` into `timing_median`, `timing_mad`, `timing_n_reps`,
  `timing_warmup`, `timing_thread_count`, blank on a `None` timing (the
  common case, since `measure_timing` defaults to `False`) or an
  `UNSUPPORTED` row.
- `trust_bench/reporting/html_report.py`: `build_html_report` rewritten
  into an explicit three-tier structure. Tier 1 (`parity_scatter`,
  `capability_frontier`, `capability_matrix`) renders first, each with a
  one-line caption. Every other study renders under one of three theme
  headings (*Common problems*, *Difficulty axes*, *Special capabilities*),
  table plus plot (where a plot exists) plus caption; a study absent from
  `output_dir` (skipped upstream for a coverage gap, or excluded via
  `--skip-slow`) is omitted without raising. `baseline`'s and
  `large_residual`'s basin-of-attraction-rate tables render as an
  auxiliary table inside their own study's section rather than being
  dropped. Status values render as coloured pills via
  `colors.status_color()`. A fixed backend-colour legend, built from
  `colors.BACKEND_COLORS` directly, is always shown regardless of which
  backends a given report's data actually contains. Timing columns
  collapse into one "median ± mad ms" column (raw seconds converted to
  milliseconds); a caveat that this is a single-machine, single-snapshot
  reading (no bare cross-language speed claim) is shown once, only when
  at least one rendered study has real timing data.

## Cycle 2: design revision

Before merge, a rendered preview of Cycle 1's report was compared against an
earlier design experiment for the same report. The experiment's structure
was preferred: a hero header, per-theme intro sentences, descriptive study
headings, and backend colour carried into table cells rather than only into
plots and the legend.

### Tests

`tests/reporting/test_html_report.py` extended: headline artefacts
(`parity_scatter`, `capability_frontier`, `capability_matrix`) show only
their plot and caption, no raw table; each theme heading is followed by its
`TIER_2_INTROS` sentence before its member studies; a `backend` column's
values render inside `class="backend-cell"`, coloured via
`colors.backend_color()`, matching the colour already used in plots and the
legend. Existing heading-position assertions updated to anchor on the new
descriptive titles (`TITLES`) instead of the raw study name.

### Implementation

`trust_bench/reporting/html_report.py`:

- A hero section (`eyebrow`/`h1`/`lede`) states what the report answers,
  followed by a combined legend: one coloured dot per backend, one coloured
  pill per status group (`CONVERGED`, `MAX_ITER / STALLED`,
  `FAILED / DIVERGED / ERROR`, `UNSUPPORTED`).
- Each study's heading is now a descriptive title (`TITLES`) rather than its
  raw registry/filename key (e.g. "Large residual (LM vs Newton)" instead of
  `large_residual`).
- Each theme (`TIER_2_INTROS`) gets a one-line intro sentence under its `h2`
  explaining what that group of studies is testing.
- Tier 1 renders only the plot and caption for each headline artefact; the
  underlying row-by-row table isn't shown there (the point of a headline
  artefact is the visual read, not the table).
- A table's `backend` column values render inside `class="backend-cell"`
  with an inline colour from `colors.backend_color()`, so a backend reads as
  the same colour in a table cell as it does in a plot line or the legend.
