# PR notes: Issue #97 - Add capability-study plots now that real trust-apl data exists

## Summary

Refines #73, which deliberately deferred choosing plots until Phase 8
(APL backend) landed and there was real `trust` data to compare
against. That data now exists: `ill_conditioning`, `dimensionality`,
`scaling`, and (since #96) `robust_loss` all show `trust-apl`-specific
behaviour currently visible only by reading raw CSV rows. This adds a
plot for each, plus a heatmap rendering of the capability matrix
(Section 9.1's own mockup table, currently only ever rendered as a
flat CSV). `bounded` deliberately excluded per the issue's own scope:
its data is categorical (scenario × method), not a natural sweep-plot.

**Design notes**

*`plot_metric_vs_sweep` is extended, not replaced or duplicated per
study.* `group` already worked with a single column name via pandas'
own `groupby`; accepting a list of column names needs no new
groupby logic, only a nicer combined label than `str()` on a tuple
(`_group_label` joins with `/`). `status_col` is optional and defaults
to `None`, so `large_residual`'s existing plot call (no status
distinction needed there) is completely unaffected - confirmed the
existing `test_plot_metric_vs_sweep_draws_one_line_per_group`-style
tests still pass unchanged.

*The non-`CONVERGED` overlay draws the full line through every point
regardless of status, then adds a hollow marker on top for the
non-`CONVERGED` ones*, rather than splitting the line into solid/dashed
segments or dropping non-`CONVERGED` points from the trend entirely:
this keeps the sweep's overall shape visible (including where it
breaks down) while still flagging exactly which points didn't honestly
converge, without the added complexity of per-segment line styling.

*Capability-matrix cell categories: `"agree"` (declared and measured
match, either both `True` or both `False`), `"declared-only"`
(`declared=True`, `measured=False` - claimed but not confirmed
working), `"disagree"` (`declared=False`, `measured=True` - working
despite not being declared, the rarer/more surprising direction).*
Matches the acceptance criterion's three named categories literally;
verified directly against the real `derive_matrix()` output (currently
all-`"agree"` for both registered backends, confirming the categoriser
doesn't misclassify the common case).

*`html_report.py` needed no changes.* `build_html_report` already
discovers every `.png`/`.csv` already written to the output directory
and embeds it automatically; a plot reaching `report.html` is a direct
consequence of `cli.py` writing the file to the same directory as its
study's table, not a separate wiring path.
