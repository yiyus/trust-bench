# PR notes: Issue #43 - CLI compare command

Part of #41 (Epic: Phase 10, Longitudinal tooling).

## Acceptance criteria

- `trust-bench compare <baseline> <candidate>` reports both classifications with full provenance

Scope clarification (issue #43 comment): "reports" means the full generated
HTML report, with the relevant graphs, not text-only CLI output - the
comparison folds into the same `report.html` `trust-bench report --html`
already produces, alongside whatever study artefacts are present.

## Cycle 1: RED

Three surfaces, three test files, all driving out code that does not exist
yet:

`tests/reporting/test_compare.py` (extends the #42 file) drives out two new
functions in `trust_bench/reporting/compare.py`:

- `compare_with_provenance(baseline, candidate, ...)`: `compare()`'s own
  classification, plus every `EnvProvenance` field for both sides, prefixed
  `baseline_`/`candidate_` - the "full provenance" the acceptance criterion
  asks for. `compare()` itself (#42) is unchanged; this is additive.
- `classification_counts(compared, group_column="backend")`: counts of each
  classification per backend, feeding the report's graph.

`tests/reporting/test_html_report.py` (extends the #42/#135 file) drives out
a "Longitudinal comparison" section in `build_html_report`: rendered when
`compare.csv` (and, where present, `compare.png`) exist in the output
directory, carrying the full-provenance table and a graph, omitted (like
every other artefact) when absent.

`tests/integration/test_cli.py` (extends the existing file) drives out:

- A `compare` subparser: positional `baseline`/`candidate` paths,
  `--output-dir` (default `reports`), `--html`.
- `run_compare(baseline_path, candidate_path, output_dir, html=False)`:
  loads both `results/*.jsonl` files (`storage.load`), writes
  `compare.csv`/`compare.png`, and - only with `--html` - (re)writes
  `report.html` via the existing `build_html_report`/`save_html_report`,
  the same path `trust-bench report --html` uses, so a comparison run folds
  into whatever report already exists in that directory rather than
  building a second one.
- `main`'s `compare` branch wiring the above through the real CLI entry
  point end to end.
